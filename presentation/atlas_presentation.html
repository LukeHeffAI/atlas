The Problem

Prior work uses a single scalar α for the entire task vector:

θ = θ0 + α · τ

All parameter blocks treated equally —
but they have different roles per task!

Quick background: A task vector is simply the difference between a fine-tuned model and its pretrained initialization. Ilharco et al. (2022) showed you can do arithmetic on these vectors — add them to combine tasks, negate to remove knowledge. But the standard approach uses a single scalar alpha to scale the entire vector. This is isotropic — every layer, every attention head, every parameter block gets the same weight. Intuitively that's suboptimal: attention layers and MLP layers play very different roles for different tasks. Implementation: see src/task_vectors.py — NonLinearTaskVector and LinearizedTaskVector classes support all arithmetic operations.

aTLAS: Anisotropic Scaling

Key idea: Learn a different coefficient per parameter block

θ = θ0 + ∑i Λi τi    where Λi = diag(αi,1, …, αi,B)

Variants: Blockwise (1 coef/block)  |  Partitioned (aTLAS × K: K coefs/block)  |  Linearized (convex optimization via Taylor approx.)

Here's the core contribution. Instead of one scalar, aTLAS learns a diagonal matrix Lambda per task vector — effectively one coefficient per parameter block. For ViT-B/32 that's about 97 blocks, so roughly 100 learnable parameters per task. Compare that to millions for full fine-tuning. This exploits the low intrinsic dimensionality of pre-trained models. The bar chart illustrates the difference: isotropic scaling applies the same weight everywhere, while anisotropic scaling lets the model emphasize MLP layers over attention keys, for example — whatever the task demands. We also have aTLAS x K, where each block is further partitioned into K sub-groups for more expressiveness (src/composition.py, see the partition parameter). For the linearized variant, we use a first-order Taylor approximation (src/linearize.py) which makes the coefficient optimization convex. Core implementation: src/composition.py — WeightedImageEncoder class, line 52.

Meta-Training

  • Episode-based: sample task → predict coefficients from text → evaluate → backprop
  • Train on diverse tasks (CIFAR10, EuroSAT, DTD, ...)
  • Validate on held-out tasks (Caltech101, Flowers102)

Text Sources

  • Manual — human-written descriptions
  • Generated — GPT-4o / Claude
  • Templates — CLIP-style prompts

+ Synthetic images via Stable Diffusion / DALL-E

This is our new extension beyond the original paper. The question: can we skip the need for task-specific images entirely? The answer is a hypernetwork that maps text descriptions to aTLAS coefficients. The pipeline: text descriptions go through a frozen CLIP text encoder, then an MLP predicts the blockwise coefficients, which are applied to compose task vectors. Meta-training uses an episode-based approach — each episode samples a random task, predicts coefficients from its text descriptions, evaluates on a validation split, and backpropagates the loss through the hypernetwork. We support three text sources: - Manual descriptions (data/text_descriptions/manual/) - LLM-generated (python -m src.text_descriptions.generators --llm gpt4o) - CLIP-style templates (automatic) Plus a complementary synthetic data approach using Stable Diffusion or DALL-E to generate images from text, then creating synthetic task vectors. Key files: - Hypernetwork: src/hypernetworks/text_to_coef.py - Meta-training: src/learn_text_to_coef.py - Evaluation: src/eval_text_adaptation.py - Text loaders: src/text_descriptions/loaders.py - T2I backends: src/text2image/ To meta-train: python src/learn_text_to_coef.py \ --model ViT-B-32 --hypernetwork-arch medium \ --meta-train-datasets CIFAR10,EuroSAT,DTD,GTSRB,SVHN,Food101 \ --meta-val-datasets Caltech101,Flowers102 \ --meta-epochs 50 --episodes-per-epoch 10 --blockwise-coef To evaluate: python src/eval_text_adaptation.py \ --model ViT-B-32 --dataset Flowers102 --approach hypernetwork \ --hypernetwork-checkpoint checkpoints/ViT-B-32/hypernetworks/text_to_coef/meta_trained.pt For ablation analysis: - python scripts/analyze_hypernetwork_size.py (architecture comparison) - python scripts/analyze_text_ablation.py (text source comparison) - python scripts/analyze_synthetic_quality.py (synthetic image quality)

Research Directions

HIGH

Ablation Studies

Text sources, T2I backends, hypernetwork architectures. Systematic benchmarking across all configurations.

HIGH

Multi-Modal Hypernetwork

Text + few example images → coefficients. Bridges zero-shot and few-shot adaptation.

MED

Full LoRA Prediction

Predict weight matrices (not just scalars). Higher expressiveness for complex tasks.

MED

Cross-Model Transfer

Meta-train on ViT-B/32, transfer to ViT-L/14 or RN50. Architecture-agnostic coefficient prediction.

MED

Domain Adaptation

Medical imaging, satellite, microscopy. Domain-specific text encoders and synthetic data pipelines.

LATER

Task Composition

Compose text descriptions: "sparrow" + "brown" + "urban" → house sparrow. Fine-grained concept composition.

Here's where we're headed. Full details are in EXTENSION_ROADMAP.md. HIGH PRIORITY (next 3 months): 1. Ablation studies — we need systematic comparison of text sources (manual vs LLM-generated vs templates), T2I backends (SD vs DALL-E), and hypernetwork architectures (small/medium/large). Analysis scripts already exist in scripts/. 2. Multi-modal hypernetwork — the most exciting near-term direction. Instead of text-only, combine text descriptions with a few example images. This bridges the gap between our zero-shot text approach and standard few-shot learning. Expected +15-25% improvement on few-shot scenarios. Planned file: src/hypernetworks/multimodal_to_coef.py MEDIUM PRIORITY (6 months): 3. Full LoRA prediction — instead of predicting scalar coefficients, predict full low-rank weight matrices. Much more expressive but harder to train. Expected +10-20% on complex tasks. Planned file: src/hypernetworks/text_to_lora.py 4. Cross-model generalization — can we train the hypernetwork on ViT-B/32 and have it transfer to completely different architectures? This requires an architecture-agnostic representation of coefficient importance. 5. Domain adaptation — applying aTLAS to specialized domains like medical imaging (privacy constraints, fine-grained), satellite (large images, temporal), and microscopy (novel organisms, tiny datasets). LOWER PRIORITY (1 year): 6. Task composition — compose multiple text concepts for fine-grained tasks. E.g., combine "bird type", "color", "habitat" descriptions to target specific species. Uses compositional text encoders.

Summary

NeurIPS 2024

Learned anisotropic scaling for task vector composition. ~100 parameters per task. Less interference, better data efficiency across negation, addition, few-shot, and TTA.

Text-Based Extension

Hypernetwork predicts aTLAS coefficients from text descriptions alone. Meta-learned across diverse tasks. Zero images required for adaptation.

What's Next

Multi-modal fusion (text + images), LoRA prediction, cross-architecture transfer, and domain-specific adaptation for medical/satellite/microscopy.

Questions & Discussion

To wrap up: 1. The original NeurIPS paper showed that learning different coefficients per parameter block — anisotropic scaling — is a simple but powerful idea for task vector composition. Only ~100 learnable parameters, works across four different settings. 2. Our extension removes the need for task-specific images entirely by using a hypernetwork to predict coefficients from text descriptions. 3. The roadmap has exciting directions — multi-modal fusion is the most promising near-term, and LoRA prediction could be a significant step up in expressiveness. Happy to take questions. Some discussion prompts: - How well do you think the text-only approach can compete with few-shot? - Which future direction seems most impactful? - Should we prioritize LoRA prediction or multi-modal fusion? - Thoughts on domain adaptation — which domain should we target first?
