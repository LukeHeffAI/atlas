# 03 — Few-Shot Adaptation

## Scientific Question

Given a pretrained CLIP model and a library of task vectors from diverse datasets, can we adapt to a *new* target task using only a handful of labeled examples (1–16 shots)? Instead of fine-tuning from scratch, aTLAS learns how to **reweight existing task vectors** to compose a model suited for the target task, treating each task vector as a reusable "knowledge module."

This is the primary practical experiment of the aTLAS framework and the main result reported in the NeurIPS 2024 paper.

## Method Overview

For each target dataset, we construct a task vector library from **all other datasets** (leave-one-out). Given $K$ labeled examples per class from the target dataset, we learn blockwise coefficients $\{\alpha_1, \ldots, \alpha_N\}$ such that:

$$\theta_{\text{adapted}} = \theta_{\text{pretrained}} + \sum_{i=1}^{N} \alpha_i \odot \tau_i$$

The model is trained to classify the $K$-shot support set correctly (cross-entropy loss), but only the coefficients are updated — the base model and task vectors remain frozen. This makes training extremely fast (seconds to minutes) and parameter-efficient (typically a few hundred coefficients for blockwise mode).

The experiment supports two important extensions:

1. **Adapter integration**: After learning aTLAS coefficients, an adapter (TIP-Adapter or LP++) can be applied on top for additional gains. The adapter operates on the composed model's features, providing a complementary form of adaptation.

2. **Synthetic data**: Instead of real few-shot samples, the model can be trained on synthetic images generated by text-to-image models (see [07-text-to-coefficient.md](07-text-to-coefficient.md)), enabling adaptation when no real data is available.

## Key Implementation Details

**Core script**: `src/learn_few_shots.py`

**Key classes**:
- `WeightedImageEncoder` (`src/composition.py`) — composition model
- `NonLinearTaskVector` / `LinearizedTaskVector` (`src/task_vectors.py`) — task vector loading

**Default hyperparameters**:
| Parameter | Value |
|-----------|-------|
| Learning rate | 1e-1 |
| Batch size | 128 (64 for ViT-L-14) |
| Epochs | 10 |
| Coefficient type | Blockwise (with `--blockwise-coef`) |
| Task vector pool | 21 datasets (all except target) |

**Key arguments**:
| Flag | Description |
|------|-------------|
| `--subsample` | Shot count(s): `4` for single, `1,2,4,8,16` for batch |
| `--blockwise-coef` | Per-block coefficients (recommended) |
| `--adapter` | Optional adapter: `tip`, `lpp`, `tip_cot` |
| `--task-vector-source` | `real` (default), `synthetic`, or `mixed` |
| `--retry-failed` | Re-run only datasets that failed in a prior run |

**Efficiency note**: When passing comma-separated shot counts (e.g., `--subsample 1,2,4,8,16`), the script loads checkpoints once and iterates over all shot settings per dataset, avoiding redundant I/O.

## How to Run

### Prerequisites
- Fine-tuned checkpoints for all 22 datasets
- Classification heads saved per dataset

### Basic aTLAS (all shot settings)
```bash
MODEL=ViT-B-32
python src/learn_few_shots.py --model $MODEL --blockwise-coef --subsample 1,2,4,8,16
```

### Single shot setting
```bash
python src/learn_few_shots.py --model $MODEL --blockwise-coef --subsample 4
```

### With adapters
```bash
# TIP-Adapter on top of aTLAS
python src/learn_few_shots.py --model $MODEL --blockwise-coef --subsample 1,2,4,8,16 --adapter tip

# LP++ on top of aTLAS
python src/learn_few_shots.py --model $MODEL --blockwise-coef --subsample 1,2,4,8,16 --adapter lpp
```

### With synthetic data
```bash
python src/learn_few_shots.py --model $MODEL --blockwise-coef --subsample 1,2,4,8,16 \
    --task-vector-source synthetic \
    --synthetic-data-location data/synthetic_images \
    --t2i-backend stable_diffusion
```

### Retry failed datasets
```bash
python src/learn_few_shots.py --model $MODEL --blockwise-coef --subsample 4 --retry-failed
```

## Expected Outputs

Results saved under `checkpoints/<MODEL>/` or `results/<MODEL>/`:

| File | Description |
|------|-------------|
| `learned_composition_{K}_shot.pt` | Best coefficients for K-shot setting |
| `learned_composition.json` | Nested results by shot count and dataset |
| `failed_evaluations.json` | Datasets that failed (for `--retry-failed`) |

**Result format** (in `learned_composition.json`):
```json
{
  "4": {
    "Cars": {"val_acc": 0.72, "test_acc": 0.71},
    "DTD": {"val_acc": 0.65, "test_acc": 0.63},
    ...
  },
  "16": { ... }
}
```

**What "good" looks like** (ViT-B-32, blockwise coefficients):
| Shots | Avg. Accuracy (approx.) |
|-------|------------------------|
| 1 | 55–60% |
| 4 | 62–67% |
| 16 | 68–73% |

These should significantly exceed zero-shot CLIP and approach (but not reach) full fine-tuning performance.

## Common Issues

- **Missing task vectors**: If a fine-tuned checkpoint is missing for any of the 21 source datasets, the script will error. Ensure all datasets are fine-tuned first.
- **Small datasets with high shot counts**: Some datasets have few samples per class. The script handles oversampling automatically, but results may have higher variance.
- **Adapter memory**: TIP adapter caches all support features, which can be memory-intensive for high shot counts on large datasets. Monitor GPU memory.
