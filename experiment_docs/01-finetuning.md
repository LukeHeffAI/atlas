# 01 — Fine-Tuning: Creating Task Vectors

## Scientific Question

How do we extract reusable, composable knowledge representations from pretrained vision-language models? The answer lies in **task vectors** — the element-wise difference between a model fine-tuned on a specific task and the original pretrained weights. If fine-tuning moves the model toward a task-specific solution, then the displacement in weight space encodes the "knowledge" gained. This experiment creates those task vectors, which serve as the raw material for all downstream aTLAS experiments.

## Method Overview

Starting from a pretrained CLIP model (e.g., ViT-B-32 from OpenAI), we fine-tune the image encoder on each of the 22 target datasets independently. The fine-tuning follows standard supervised classification: the image encoder feeds into a linear classification head, and we minimize cross-entropy loss on the training split.

The **task vector** for dataset $D$ is then:

$$\tau_D = \theta_{\text{finetuned}}^D - \theta_{\text{pretrained}}$$

Two fine-tuning modes are supported:

- **Standard**: Regular backpropagation through the full model.
- **Linearized**: Fine-tunes a first-order Taylor approximation of the model around the pretrained weights, using `functorch.jvp` for Jacobian-vector products. This enables "tangent task arithmetic" with improved compositionality in some settings.

A third variant fine-tunes on **synthetic images** generated by text-to-image models (see [07-text-to-coefficient.md](07-text-to-coefficient.md)), producing synthetic task vectors that approximate real ones without requiring any real task-specific data.

## Key Implementation Details

**Core script**: `src/finetune.py`

**Key classes**:
- `ImageEncoder` (`src/modeling.py`) — wraps OpenCLIP models, provides train/val preprocessing
- `ImageClassifier` (`src/modeling.py`) — encoder + linear classification head
- `LinearizedModel` (`src/linearize.py`) — first-order Taylor approximation wrapper
- `NonLinearTaskVector` / `LinearizedTaskVector` (`src/task_vectors.py`) — task vector extraction and arithmetic

**Default hyperparameters**:
| Parameter | Value |
|-----------|-------|
| Learning rate | 1e-3 |
| Weight decay | 0.1 |
| Batch size | 128 |
| Epochs | 10–20 (dataset-dependent) |
| Warmup | 500 steps |
| Label smoothing | 0.0 |
| Precision | AMP (mixed precision) |

## How to Run

### Prerequisites
- Datasets prepared per `DATASETS.md`
- Pretrained CLIP checkpoints available (auto-downloaded by OpenCLIP)

### Standard fine-tuning on a single dataset
```bash
MODEL=ViT-B-32
python src/finetune.py --model $MODEL --train-dataset CarsVal --epochs 10
```

### Linearized fine-tuning
```bash
python src/finetune.py --model $MODEL --train-dataset CarsVal --epochs 10 --finetuning-mode linear
```

### Fine-tuning on synthetic images
```bash
python src/finetune.py --model $MODEL --train-dataset CIFAR10Val \
    --task-vector-source synthetic \
    --synthetic-data-location data/synthetic_images \
    --t2i-backend stable_diffusion
```

### Fine-tune all 22 datasets (full pipeline)
```bash
MODEL=ViT-B-32
for DATASET in Cars DTD EuroSAT GTSRB MNIST RESISC45 SVHN CIFAR10 CIFAR100 \
    STL10 Food101 Caltech101 Caltech256 FGVCAircraft Flowers102 OxfordIIITPet \
    CUB200 PascalVOC Country211 UCF101 SUN397 ImageNet; do
    python src/finetune.py --model $MODEL --train-dataset ${DATASET}Val --epochs 20
done
```

## Expected Outputs

Per dataset, saved under `checkpoints/<MODEL>/<DATASET>Val/`:

| File | Description |
|------|-------------|
| `zeroshot.pt` | Pretrained CLIP encoder weights (saved once) |
| `finetuned.pt` | Fine-tuned encoder weights |
| `linear_zeroshot.pt` | Linearized pretrained weights (if `--finetuning-mode linear`) |
| `linear_finetuned.pt` | Linearized fine-tuned weights |
| `synthetic_<backend>_finetuned.pt` | Fine-tuned on synthetic images |

Classification heads are saved separately as `head_<DATASET>.pt`.

**Sanity check**: Fine-tuned accuracy on the target dataset should significantly exceed zero-shot accuracy (typically +10–40% depending on dataset difficulty and domain gap from CLIP's pretraining).

## Common Issues

- **Out of memory**: ViT-L-14 may require reducing batch size to 64 or using gradient accumulation.
- **Dataset not found**: Ensure `--data-location` points to the correct root and that datasets are prepared per `DATASETS.md`. Several datasets (Cars, ImageNet, UCF101) require manual download.
- **Linearized mode slow**: The JVP computation in linearized fine-tuning is ~2–3x slower than standard. This is expected.
